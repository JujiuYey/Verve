# 00 · 架构总览

## 分层

```text
前端 React SPA
  │  HTTP / SSE(/api/*)
后端 Go + Fiber
  ├ router        路由注册(/api/learning/*)
  ├ handler       参数校验、调 service、统一响应 / SSE 输出
  ├ service       业务逻辑、agent 编排调用、事务
  ├ repository    bun 数据访问(FindPage/FindOne/Create/Update/Delete)
  ├ model(agent) eino adk agent 定义 + Runner
  └ tools         eino 工具(读写 learning 数据)
数据  Postgres(bun) / Redis(asynq) / 模型(ai_model_config)
```

## 请求流(两类)

**普通 CRUD**(创建目标、查画像、列表):

```text
前端 → Fiber 路由 → AuthMiddleware → handler → service → repository → Postgres
                                                      ↘ 统一 Response[T] 返回
```

**Agent 流式**(陪练对话、生成路线):

```text
前端(fetch stream)→ handler → service 构造 adk Runner
  → Runner.Query → AsyncIterator[AgentEvent]
  → handler 用 SetBodyStreamWriter 把事件转 SSE(data: {...}\n\n)
  → 前端 streamdown 渲染
  ↘ 过程落 learning_messages / learning_exercises
```

## 跨切面约定

| 切面 | 做法 |
|---|---|
| 认证 | `protected.Use(AuthMiddleware())`;handler 内 `c.Locals("user_id").(string)` |
| 响应 | `response.SuccessCtx / PaginateCtx / BadRequestCtx / UnauthorizedCtx / ...` |
| 错误 | 校验失败→400;未登录→401;访问他人资源→403;不存在→404;内部→500 |
| 数据隔离 | 所有 learning 查询带 `user_id` 条件,写操作校验资源归属 |
| 流式 | SSE,事件结构与现有 `chat.go` 一致(`type`/`content`/`agent`) |
| ID | `varchar(32)`,沿用现有生成方式 |
| 装配 | `DatabaseService` 持有各 learning repository;`router.go` 受保护组注册 learning 路由 |

## 前端对接(简述)

- 普通接口:axios + TanStack Query。
- 流式:fetch + ReadableStream 解析 `data: {...}`,复用现有 `components/ai-elements` 的 SSE 消费 + `streamdown` 渲染。
- 路由:TanStack Router,挂在 `routes/_layout`(需登录布局)下。
- `(待深化)`:前端方案细化(状态、组件拆分)可另开篇。
