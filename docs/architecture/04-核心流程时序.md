# 04 · 核心流程时序

> 参与者:前端 / handler / service(含 orchestrator)/ agent(adk Runner)/ repository(bun)/ Postgres。

## 流程一:创建目标 → 生成路线

```mermaid
sequenceDiagram
  participant FE as 前端
  participant H as handler
  participant S as service(Planner)
  participant A as Planner(adk)
  participant R as repository
  FE->>H: POST /api/learning/goal {title}
  H->>S: CreateGoal(userID, title)
  S->>R: insert learning_goals
  S->>A: Runner.Query(规划 prompt)
  A-->>S: 结构化路线(流式 / 一次)
  S->>R: insert learning_paths + objectives(事务)
  S-->>H: goal + path
  H-->>FE: SSE 过程 + done(或同步 JSON)
```

## 流程二:上一节课(核心闭环)

```mermaid
sequenceDiagram
  participant FE as 前端
  participant H as handler
  participant O as orchestrator(service)
  participant T as Tutor(adk)
  participant E as Examiner(adk)
  participant R as repository
  FE->>H: POST /session {objective_id}
  H->>R: insert learning_sessions
  FE->>H: POST /session/:id/chat (SSE)
  H->>O: 取上下文(objective/profile/history)
  O->>T: Runner.Query(诊断→讲解→练习)
  T-->>H: 事件流(phase=diagnose/explain/practice)
  H-->>FE: SSE(stream_chunk/message/exercise)
  H->>R: 落 learning_messages
  FE->>H: POST /session/:id/exercise {type,answer}
  H->>E: 判定
  E->>R: insert learning_exercises + update mastery
  E-->>H: verdict + feedback
  H-->>FE: 判定结果
  FE->>H: POST /session/:id/complete
  H->>E: 汇总本节
  E->>R: update objective/profile/journal + 推进 current_objective
  H-->>FE: summary + next_objective
```

## 流程三:次日继续

```mermaid
sequenceDiagram
  participant FE as 前端
  participant H as handler
  participant R as repository
  FE->>H: GET /api/learning/continue
  H->>R: 取最近 active path.current_objective
  R-->>H: objective
  H-->>FE: {goal_id, objective_id, title}
```

> 说明:流程二里 chat 与 exercise 是分开的请求 —— 陪练走 SSE 流式,验证提交走普通 POST 同步返回判定;两者都落库,`complete` 收尾时统一写画像 / 日志并推进进度。
