> learning 模块沿用现有 DDD 分层(handler → service → repository → db)。本篇讲**每个功能怎么落地**;接口契约见 [02](02-API接口设计.md),agent 细节见 [03](03-Agent编排设计.md)。

## 模块装配(对齐现有 main.go / router.go)

- `DatabaseService` 增加 learning 各 repository:`dbService.Goals / Paths / Objectives / Sessions / Messages / Exercises / Profiles / Journals`。
- `router.go` 受保护组里加一行:`learning_router.SetupLearningRoutes(protected.Group("/"), dbService, ...)`。
- agent 懒加载,参考现有 `chat.go` 的 `ensureAgent`(首次用时初始化 Runner)。

## 各功能的实现路径

### F-02 / F-04 · 创建目标 + 生成路线(Planner)

```text
POST /api/learning/goal { title }
  1. handler 校验 → service.CreateGoal:建 learning_goals(source=text, user_id)
  2. 触发 Planner:构造 Planner agent(adk),输入 title + 用户已有画像背景
  3. Planner 产出结构化路线 → 落 learning_paths(overview) + 批量 learning_objectives
  4. 设 paths.current_objective_id = 第一个 objective
  5. 返回目标 + 路线
```

- **生成方式**:MVP 建议**流式同步**(SSE 展示"正在规划…"),实现简单;量大可改 asynq 异步 + 轮询。
- **一致性**:目标 + 路线 + objectives 在**一个事务**里,失败回滚,不留半成品。

### F-03 / F-05 · 目标列表 / 详情 / 进度

- `GET /api/learning/goal/page` → `repository.FindPage` + `PaginateCtx`。
- `GET /api/learning/goal/:id` → 目标 + path + objectives(bun relation 或分查组装)。
- **进度** = `objectives(completed) / objectives(total)`,可在查询时聚合。

### F-06 · 陪练教学单节(Orchestrator + Tutor)

```text
POST /api/learning/session            建 learning_sessions(active, 关联 objective)
POST /api/learning/session/:id/chat   (SSE)
  1. service 取上下文:objective、profile、本 session 历史 messages
  2. 构造 Orchestrator/Tutor 的 adk Runner(见 03)
  3. Runner.Query → 事件流 → SetBodyStreamWriter 转 SSE
  4. 每条消息落 learning_messages(agent_type, role, content, token)
```

- 教学铁律(每次一个认知点)通过 prompt + orchestrator 控制(见 03)。

### F-07 · 学习验证(Examiner)

```text
POST /api/learning/session/:id/exercise { type, user_answer }
  1. service 调 Examiner agent 判定
  2. 落 learning_exercises(prompt, user_answer, verdict, mastery_after, feedback)
  3. 回写 learning_objectives.mastery_level
  4. 返回 verdict + feedback(可同步;也可流式)
```

- 验证类型:`explain / choice / cloze / paste_output / code_snippet`(对应 03 的 Examiner 工具)。

### F-08 · 学习画像

- `GET /api/learning/goal/:id/profile` → 读 `learning_profiles`。
- **写**:Examiner 在本节结束更新 `completed_topics / weak_points / next_goal`(通过工具写库)。

### F-09 · 继续上次 / 今日推荐

- `GET /api/learning/continue` → 取用户最近 active path 的 `current_objective_id`,返回"下一节"入口信息。

### F-10 · 学习日志

- `GET /api/learning/journal/page` → 按天分页。
- **写**:本节结束 Examiner 生成草稿写 `learning_journals`(MVP 自动写;确认式见 ../product/07 Q5)。

### F-01 · 账户

- 复用现有 auth / user;精简掉 role/department(见 [../product/09-表结构设计.md](../product/09-表结构设计.md))。

## 事务与一致性要点

- **建目标 + 路线 + objectives**:单事务。
- **本节结束**(更新 objective + profile + journal):service 层事务包裹关键写,避免画像/日志写一半。
- **agent 调用失败**:不阻塞已写入的用户消息;向前端发 `error` 事件(同 chat.go)。
