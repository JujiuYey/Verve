# 03 · 用户旅程与流程

## 首次使用旅程(端到端)

```text
注册 / 登录
  → 新建目标(一句话)
  → 等待生成路线(有进度反馈)
  → 看到学习路线
  → 开始第一节:诊断 → 讲解 → 练习 → 验证
  → 看到"通过 / 反馈"
  → 画像 + 日志更新
  → 推荐下一步
```

**关键情绪点**

- **生成路线**要快、要显得靠谱 —— 这是第一印象,崩了就流失;
- 第一次**验证通过**是 aha moment:用户第一次真切感到"它在盯我、确认我学会了"。

## 单节学习闭环(核心)

```text
进入 session → Orchestrator 调度
  → Tutor:诊断问题 → 简短讲解 → 小练习 → 要求验证
  → 用户提交(解释 / 代码 / 粘贴运行结果)
  → Examiner 判定 → 写入验证记录
  → 本节小结 → 更新 小目标 / 画像 / 日志
  → 推荐下一步
```
(对应 [../product/08-产品地图.md](../product/08-产品地图.md) 流程二、[../product/04-Agent系统设计.md](../product/04-Agent系统设计.md))

## 次日回访旅程

```text
登录 → 学习总览看到「继续上次」+ 今日推荐 → 一键进入下一节
```

## 异常 / 边界流

| 场景 | 处理 |
|---|---|
| 路线生成失败 / 超时 | 明确提示 + 可重试,不留半成品目标 |
| 用户答非所问 / 卡住 | Tutor 引导,不强行通过 |
| 用户中途放弃本节 | session 标 `abandoned`,下次可恢复 |
| 验证反复不通过 | 降难度 / 换讲法,标记为薄弱点 |
