> 页面 / 路由完整列表见 [../product/08-产品地图.md](../product/08-产品地图.md)。本篇给关键页面的**低保真线框**与交互说明(线框含中文,框线对齐为示意,看结构与元素即可)。

## 信息架构

```text
登录 / 注册
└ 主框架(侧边栏 + 内容区)
   ├ 学习总览        /

   ├ 学习画像        /goals/:id/profile
   ├ 学习日志        /journal
   └ 设置            /settings
```

## 线框图(低保真)

### 学习总览

```text
┌─ [Logo] ───────────────────────── 学习总览 ──── [头像▾] ─┐
│ 侧边栏      │  今天学什么                                │
│ ▸学习总览   │   ┌────────────────────────┐  [▶ 继续上次] │
│ ▸我的目标   │   │ 当前:goroutine 调度    │              │
│   ·Go       │   └────────────────────────┘              │
│   ·Rust     │  我的目标                                  │
│ ▸学习日志   │   ┌────────┐ ┌────────┐ ┌──────────┐       │
│ ▸设置       │   │Go   60%│ │Rust 20%│ │ + 新建    │      │
│             │   └────────┘ └────────┘ └──────────┘       │
└─────────────┴────────────────────────────────────────────┘
```
- 元素:今日推荐(继续上次)、目标卡片(带进度环)、新建入口。
- 目的:回访用户**一眼看到"接着学什么"**,降低重新进入成本。

### 新建目标

```text
┌──────────────────────────────────────────────┐
│  你想学什么?                                  │
│  ┌──────────────────────────────────────────┐ │
│  │ 例如:我要学 Go 的并发                     │ │
│  └──────────────────────────────────────────┘ │
│  快捷示例:[Rust 所有权] [K8s 入门] [SQL 优化] │
│                            [ 生成学习路线 → ]  │
└──────────────────────────────────────────────┘
```
- 元素:一句话输入、示例 chips(降低冷启动)、生成按钮。
- 交互:提交后进入"生成中"态(进度反馈),完成跳目标详情。

### 目标详情(学习路线)

```text
┌──────────────────────────────────────────────┐
│ Go 并发            进度 ███░░ 60%    [继续 ▶]  │
├──────────────────────────────────────────────┤
│ 阶段一 · 基础 ✓                                │
│    ✓ goroutine 是什么                          │
│    ✓ channel 基础                              │
│ 阶段二 · 进阶  ◀ 当前                          │
│    ● goroutine 调度          [ 开始这一节 ]    │
│    ○ select 多路复用                           │
│    ○ context 取消                              │
└──────────────────────────────────────────────┘
```
- 元素:进度条、阶段分组、小目标状态(✓完成 / ●当前 / ○未开始)。

### 陪练教学(核心页)

```text
┌─ goroutine 调度 ───────────────────┬─ 本节进度 ─┐
│ 🤖 先问你:N 个 goroutine 阻塞时,  │  诊断 ✓    │
│    调度器会怎么处理?               │  讲解 ✓    │
│ 🧑 我觉得……                        │  练习 ◀    │
│ 🤖 对了一半,看这个例子……          │  验证 ○    │
│ ┌── 练习 ─────────────────────────┐ │            │
│ │ 写一段:启动 3 个 goroutine 并   │ │            │
│ │ 等它们全部结束                   │ │            │
│ │ [ 代码输入… ]                    │ │            │
│ │ [ 粘贴运行结果… ]      [ 提交 ]  │ │            │
│ └──────────────────────────────────┘ │            │
└────────────────────────────────────┴────────────┘
```
- 元素:对话流(流式)、练习卡(按类型变输入控件)、右栏本节进度。
- 目的:右栏让用户**随时知道走到"诊断→讲解→练习→验证"哪一步**,有掌控感。

### 学习画像

```text
┌──────────────────────────────────────────────┐
│ Go 并发 · 学习画像                             │
│ 当前水平:入门 → 进阶                          │
│ 已掌握:goroutine、channel 基础                │
│ 薄弱点:⚠ select 阻塞、⚠ context 传播          │
│ 下一步:context 取消传播                       │
└──────────────────────────────────────────────┘
```

## 交互原则

- **流式 + 打字机**输出;每次只呈现一个认知点(对齐教学铁律,见 ../product/03)。
- **验证输入随类型变化**:解释框 / 选项 / 代码框 / 粘贴框。
- **进度可见**:陪练页右栏常驻本节四步进度;目标页常驻整体进度。
- `(待深化)`:高保真视觉、组件态、空 / 错 / 加载态、移动端适配。
