# 01 · 用户与画像

## 目标用户

个人(to C)、**技术 / 编程自学者**。

## Persona(草案,待你校正)

### Persona A — 转方向的开发者「小赵」
- **背景**:有编程基础,想系统学一门新语言 / 领域(如 Rust、分布式、K8s);
- **痛点**:零散搜索 + 看文档,学了就忘,不确定有没有真懂;
- **期望**:有人带着按路线学、给练习、验证、盯进度。

### Persona B — 想打牢基础的初学者「小钱」
- **背景**:刚入门,知识点零碎、容易高估自己;
- **痛点**:不知道学习顺序,缺监督和反馈;
- **期望**:一步步推进、不跳跃,每天有明确小目标。

### Persona C — 带资料系统学的人「小孙」`(MVP 外)`
- **背景**:手上有文档 / 教程 / 论文,想被带着学完;
- 对应「上传文档」第二入口,启用时纳入。

## 用户动机与触发场景

- 心声:"我想学 X,但**不知道从哪开始 / 学了记不住 / 没人监督**。"
- 触发点:换工作、接新项目、想系统提升、备战面试。

## 非目标用户(MVP)

- 企业 / 团队 / 学校批量培训(to B);
- 只想随便问问、不追求"学会"的用户。

> `(待深化)`:真实用户访谈后校正 persona 与痛点优先级。
