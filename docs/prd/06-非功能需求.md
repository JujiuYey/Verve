# 06 · 非功能需求

## 安全

- 密码 **bcrypt** 存储(现有);会话用 **JWT**;受保护接口校验登录态。
- **API key(BYOK)加密存储**,不明文落库、不返回前端(现有 `ai_model_config.api_key` 已标注"需加密")。
- **越权防护**:学习数据一律按 `user_id` 隔离,接口校验资源归属。

## 隐私(to C 重点)

- 用户的学习内容、对话、画像属于**个人数据,仅本人可见**;
- 提供**删除账户 / 清空学习数据**的能力;
- LLM 调用涉及第三方处理 / 数据出境,需在隐私说明中告知。
- `(待深化)`:合规级别(地区 / 法规)与数据保留期限。

## 性能

- **路线生成**:异步 + 进度反馈,避免长时间白屏;
- **陪练流式**:首字延迟尽量低,体验顺滑;
- **查询**:列表 / 画像 / 会话走索引(见 ../product/09 的索引设计)。

## 可用性 / 可靠性

- 路线生成失败可重试、不留脏数据(半成品目标);
- session 可恢复(`abandoned` → 继续);
- 关键写入(画像 / 日志)失败不影响主对话流,可补偿。

## 可观测

- 关键操作与 agent 调用留痕(`learning_messages` + token 字段),便于排查与后续用量计量。

> 多为草案,具体目标值(延迟阈值、可用性 SLO、合规级别)`(待深化)`。
