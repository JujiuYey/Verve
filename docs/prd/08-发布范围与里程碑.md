# 08 · 发布范围与里程碑

## MVP 发布范围

- **做(P0)**:账户、一句话建目标、路线生成、路线 / 进度展示、陪练单节闭环、验证判定、画像、继续上次。
- **紧接(P1)**:学习日志、BYOK。
- **不做**:材料上传、代码沙箱、订阅计费、to B。

(对齐 [../product/06-MVP范围与路线图.md](../product/06-MVP范围与路线图.md))

## 里程碑(草案,与 product 路线图呼应)

| 阶段 | 目标 | 交付 |
|---|---|---|
| **M1 闭环跑通(MVP)** | 单目标、单节闭环可用,能验证、能写画像 / 日志 | F-01~F-09(P0) |
| **M2 留存打磨** | 日志、继续上次、今日推荐体验打磨 | F-10、体验 |
| **M3 监督质量** | 把"判断真学会"做扎实(产品壁垒) | Examiner 优化 |
| **M4 扩展与商业化** | 第二入口(上传文档)+ BYOK + 订阅 / 用量 | F-11、F-12、F-14 |

## 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 路线生成质量不稳 | 第一印象崩、流失 | Planner prompt 迭代 + 失败重试 |
| "判断真学会"不准(空泛鼓励 / 误判) | 核心价值打折 | Examiner 多手段验证 + 人工抽检 |
| LLM 成本(多 agent 烧 token) | 难以为继 | 早期预置 key / 控轮次,商业化上限额 |
| 纯概念主题验证弱 | 体验不一致 | MVP 限定技术 / 编程类(D6) |
