# 02 · 用户故事与用例

> 格式:**作为 \<角色\>,我想要 \<能力\>,以便 \<价值\>**。每条带验收标准(AC)。
> 优先级:**P0** = MVP 必须,**P1** = MVP 后紧接,**P2** = later。

## 账户

**US-01(P0)** 作为新用户,我想注册并登录,以便保存我的学习进度。
- AC:邮箱 + 密码注册;登录得到会话;未登录访问受保护页被拦截。

## 学习目标

**US-02(P0)** 作为用户,我想用一句话说出"我要学 X",以便快速开始、不用准备资料。
- AC:提交目标后,系统在合理时间内生成一条学习路线;目标出现在我的列表;生成失败有明确提示且可重试。

**US-03(P0)** 作为用户,我想看到我所有学习目标和各自进度,以便管理在学的东西。
- AC:列表展示目标 + 进度;可进入详情;可归档。

## 学习路线

**US-04(P0)** 作为用户,我想看到目标被拆成有顺序的阶段和小目标,以便知道学习路径。
- AC:路线按阶段分组展示小目标;高亮当前小目标;完成后进度推进。

## 陪练教学

**US-05(P0)** 作为用户,我想被一节节带着学、每次聚焦一个点,以便不被信息淹没。
- AC:单节遵循「诊断→讲解→练习→验证」;一次只推进一个认知点;输出流式呈现。

**US-06(P0)** 作为用户,我想随时继续上次的进度,以便每天接着学。
- AC:"继续上次"直接进入当前小目标的下一节。

## 学习验证

**US-07(P0)** 作为用户,我想通过解释 / 写代码 / 粘贴运行结果来证明我学会了,以便得到真实反馈而非空泛鼓励。
- AC:支持 explain / choice / cloze / paste_output / code_snippet;系统给出 pass / partial / fail + 反馈;掌握层级随之更新。

## 学习画像与日志

**US-08(P0)** 作为用户,我想看到我的掌握情况、薄弱点和下一步,以便了解自己学得怎样。
- AC:画像展示已掌握主题 / 薄弱点 / 下一步,随学习自动更新。

**US-09(P1)** 作为用户,我想看到每天的学习日志,以便回顾。
- AC:按天展示「学了什么 / 证据 / 薄弱点 / 下一步」。

## 设置

**US-10(P1)** 作为用户,我想配置自己的模型 / API key(BYOK),以便用自己的额度。
- AC:MVP 用系统预置;BYOK 为 P1;key 不明文展示。
