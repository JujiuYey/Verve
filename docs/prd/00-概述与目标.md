# 00 · 概述与目标

## 背景与问题

技术自学者面对的真实问题:

- 通用 AI 聊天能答疑,但**不记得你学到哪**、不监督、不验证,每次从零开始;
- 自学缺反馈与监督,容易"看过就以为会了",难以判断是否真掌握;
- 学习路线零散,不知道下一步该学什么。

## 产品定位

一个**监督式 AI 私教**:用户用一句话(或上传文档)给出学习目标,平台自动规划路线、陪练、监督验证、维护长期学习画像。详见 [../product/00-产品定位与商业形态.md](../product/00-产品定位与商业形态.md)。

## 产品目标

| 层面 | 目标 |
|---|---|
| 用户价值 | 让用户明显感到"它知道我学到哪、下一步该练什么",并真的学会 |
| MVP 目标 | 验证"监督式学习闭环"对个人技术学习者成立 |
| 业务目标 | 早期验证留存与口碑,为商业化(订阅)铺路 |

## 成功指标

**北极星指标**:每周完成 ≥1 个「验证通过的学习小目标」的活跃用户数。
> 它同时反映三件事:用户在学 + 真的学会了(验证通过)+ 持续回来。

| 类别 | 指标 | MVP 重点 | 目标值 |
|---|---|---|---|
| 激活 | 新用户完成首个学习闭环(建目标→学一节→验证通过)比例 | ⭐ | `(待定)` 建议 ≥ 40% |
| 留存 | 次日 / 7 日回访率 | ⭐ | `(待定)` |
| 学习成效 | 验证通过率、掌握层级提升速度 | ⭐ | `(待定)` |
| 参与 | 周均学习节数 / 时长 | | `(待定)` |
| 口碑 | NPS / 主动推荐率 | | `(待定)` |

> 目标值需要你按预期拍板,先标 `(待定)`;指标拆解与埋点见 [07-指标与埋点](07-指标与埋点.md)。

## 非目标(MVP 不追求)

- 覆盖任意学习主题(MVP 限技术 / 编程类);
- to B / 团队 / 学校;
- 商业化变现(早期先验证价值)。
