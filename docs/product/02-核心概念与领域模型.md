# 02 · 核心概念与领域模型

## 核心概念词汇表

| 概念 | 含义 |
|---|---|
| **LearningGoal(学习目标)** | 用户自定义的"我要学什么"。可由一句话或一批上传文档来定义。取代了初版设想里的"学科(Subject)"。 |
| **Material(学习材料)** | 用户上传的文档 / 资料,关联到某个学习目标,经解析、向量化后进入检索。 |
| **LearningPath(学习路线)** | 学习设计 agent 针对某个 LearningGoal 生成的阶段序列与小目标。 |
| **LessonSession(学习会话)** | 一节课 / 一次学习会话,围绕一个小目标展开。 |
| **LearningProfile(学习画像)** | 用户在某目标下的长期学习状态:掌握程度、已完成主题、薄弱点、下一步。 |
| **LearningJournal(学习日志)** | 每日学习记录:学了什么、有何证据、薄弱点、下一步。 |
| **AgentRun** | 一次 agent 执行的记录(输入 / 输出 / 工具调用 / 状态)。 |

## 概念关系

```text
User
 └── LearningGoal(我要学什么)
       ├── Material[](上传的资料,可选)
       ├── LearningPath(生成的路线:阶段 + 小目标)
       │     └── LessonSession[](一节节的学习会话)
       │           └── AgentRun[](每次 agent 执行)
       ├── LearningProfile(这个目标下的长期画像)
       └── LearningJournal[](每日日志)
```

## 数据模型(草案)

> 来自初版评估,去掉了 `Subject`,新增 `LearningGoal` 与 `Material`。字段为草案,随功能细化调整。

```text
User(已有 system.user)
- id / name / email / preferences

LearningGoal
- id / userId
- title            // 例:"我要学 Rust 的并发"
- source           // text(一句话) | documents(上传材料)
- description / status / createdAt

Material
- id / goalId / userId
- fileRef          // MinIO
- status           // 解析 / 向量化状态(chunk 进 Qdrant)

LearningPath
- id / userId / goalId
- stages[]         // 阶段序列
- currentStage / currentObjective

LessonSession
- id / userId / goalId
- objective / status / startedAt / endedAt

LearningProfile
- userId / goalId
- currentLevel / completedTopics / weakPoints
- verificationHabits / nextGoal

LearningJournal
- id / userId / date
- learned / evidence / weakPoints / nextStep

AgentRun(可由已有 agent_session / agent_message 演进)
- id / sessionId / agentType
- input / output / toolCalls / status
```
