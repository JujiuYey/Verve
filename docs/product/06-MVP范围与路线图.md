# 06 · MVP 范围与路线图

## MVP 目标

只验证一个能证明价值的闭环(见 [03](03-学习闭环与教学法.md)):

```text
给出目标 → 生成路线 → 开始一节 → 小练习 → 验证 → 写入状态 → 推荐下一步
```

**验证标准**:用户明显感觉"它知道我学到哪了,也知道我下一步该练什么"。

## MVP 范围(做 / 不做)

> MVP 范围已定:**一句话主题驱动 + 技术 / 编程类**(见 [07](07-决策记录与开放问题.md) 决策 D5/D6)。功能模块详见 [05](05-功能模块.md),表结构见 [09](09-表结构设计.md)。

**先做**
- 一个学习目标 → 一条路线 → 一个教学 agent → 一个监督 agent → 一份画像 → 一份日志 → 一个完整 session 闭环。
- 验证手段先用轻量的(解释 / 选择题 / 粘贴结果 / 代码片段)。

**先不做**
- 代码沙箱 / 远程执行;
- 多学习目标并行的复杂调度;
- 订阅计费(早期 BYOK / 预置 key);
- to B / 团队 / 权限体系。

## 现有脚手架复用映射

本项目从一个 AI 知识库 SaaS 脚手架(后端 module `sag-wiki`)copy 而来,很多地基现成。

| 学习平台所需 | 脚手架现状 | 动作 |
|---|---|---|
| 用户 / 认证 | system + JWT(去掉 RBAC,只留 user + auth) | 复用(精简) |
| agent 编排与运行 | eino + agent_session/message + tools | 复用为地基,新增学习 agent |
| 上传文档 → 向量化 → 检索 | file + RAG 管道 + Qdrant | 直接复用,接 Material |
| 路线 / 内容 / 笔记承载 | wiki document/folder + Tiptap | 改造 |
| 陪练 / 流式 UI | 前端 ai-elements + streamdown | 复用 |
| 长任务 / 后台 | asynq 队列 | 复用 |
| 多模型 / 成本 | model_config | 复用为 BYOK / 计费地基 |
| 学习状态(目标 / 路线 / 画像 / 日志) | 无 | 新建 |
| 学习 agent + orchestrator | 仅通用 agent 地基 | 新建 |

## 里程碑

`(待深化)` 初步分段:

1. **闭环跑通**:单目标、单节课、能验证、能写画像 / 日志。
2. **第二入口接入**:补上"上传文档 / 一句话"中尚未先做的那个。
3. **打磨监督质量**:把"判断真学会"做扎实(产品壁垒)。
4. **商业化准备**:订阅 + 用量限额。
