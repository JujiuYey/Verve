# 00 · 产品定位与商业形态

## 一句话定位

一个**监督式 AI 私教** —— 用户驱动、主题无关的自适应学习平台。

## 产品形态

用户带着**任意**学习目标进来,有两个入口:

- **上传一批文档**:把要学的资料丢进来;
- **一句话**:直接告诉 AI "我要学 X"。

然后平台的 agent 自动:理解目标 → 拆成学习路线 → 一节节费曼式陪练 → **监督验证是否真学会** → 维护长期学习画像 → 推荐下一步。

## 核心价值

它的价值不是"AI 能回答问题",而是平台**持续知道**用户:

- 正在学什么、已经学到哪一步;
- 哪些概念还不稳;
- 今天应该练什么;
- 学习结果是否通过代码 / 测试 / 解释验证;
- 下一步该进入哪个小目标。

## 差异化

和普通 AI 聊天 / 通用学习工具的根本区别:**它当你的私教,有长期记忆和监督**。它知道你学到哪、会不会、下一步该练什么,而不是每次从零开始的一问一答。

> 真正的护城河是"怎么判断用户真学会、怎么维护长期学习状态",详见 [03-学习闭环与教学法](03-学习闭环与教学法.md)。

## 目标用户

- **个人(to C),个人订阅。**
- 不做 to B(学校 / 团队 / 培训机构)进 MVP。现有脚手架带的 RBAC(department / role)在个人订阅下用不到,**直接去掉,`system` 模块只保留 user + auth**(见 [07](07-决策记录与开放问题.md) D7、[09](09-表结构设计.md));架构上不堵死后续 to B。
- 更细的用户画像与场景见 [01-用户与核心场景](01-用户与核心场景.md)。

## 商业形态与成本模型

LLM token 是这个产品的成本命门 —— 多 agent + 长对话会持续烧 token,必须从一开始就想清楚谁付费。

| 阶段 | 谁付 token | 形态 |
|---|---|---|
| 早期(自用 + 小范围验证) | 用户自带 key(BYOK)或平台预置 key 不计费 | 先验证产品价值,不急于商业化 |
| 商业化 | 平台 | 订阅制 + 用量限额(套餐分级) |

- 现有 `model_config` 模块支持多模型配置,是 BYOK / 计费这一层的地基。
- 定价细节、套餐分层 `(待深化)`。
