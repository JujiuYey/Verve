# 09 · 表结构设计(MVP)

> 这是 MVP 实际落地的**完整库表设计**(自包含)。范围:一句话主题驱动 + 技术 / 编程类。
>
> 共 **10 张表**,分三组:账户(1)、模型(1)、学习核心(8)。保留复用的现有表也一并给出完整 DDL,看完即知整个库全貌。
>
> 约定对齐现有(见 `ai_*` / `sys_*` 表):`VARCHAR(32)` 字符串主键(应用层生成)、外键 `ON DELETE CASCADE`、`created_at/updated_at` 默认当前时间、枚举用 `CHECK`、结构化字段用 `JSONB`、token 用量 `BIGINT`。

## 表总览(完整)

| 表 | 分组 | 来源 | 用途 |
|---|---|---|---|
| `sys_users` | 账户 | 现有 · 保留 | 用户账户 |
| `ai_model_config` | 模型 | 现有 · 保留 | 模型 / key 配置(MVP 系统预置) |
| `learning_goals` | 学习 | 新增 | 学习目标(一句话) |
| `learning_paths` | 学习 | 新增 | 学习路线(一个目标一条) |
| `learning_objectives` | 学习 | 新增 | 小目标(进度 / 掌握最小单位) |
| `learning_sessions` | 学习 | 新增 | 一节课会话 |
| `learning_messages` | 学习 | 新增 | 陪练对话(多 agent) |
| `learning_exercises` | 学习 | 新增 | 练习与验证 |
| `learning_profiles` | 学习 | 新增 | 学习画像(一个目标一份) |
| `learning_journals` | 学习 | 新增 | 每日学习日志 |

**已移除**(现有,但 to C 用不到,见 [07](07-决策记录与开放问题.md) D7):`sys_departments`、`sys_roles`、`sys_user_departments`、`sys_user_roles`。

## 关系图

```text
sys_users(账户根)
  └─ learning_goals(每个目标属于一个用户)
       ├─ learning_paths(1:1)── learning_objectives(1:N)
       │         └─ current_objective_id ──(软引用)── learning_objectives
       ├─ learning_sessions(1:N,挂在某个 objective 上)
       │       ├─ learning_messages(1:N)
       │       └─ learning_exercises(1:N)
       ├─ learning_profiles(1:1)
       └─ learning_journals(1:N,按天)

ai_model_config(系统级,独立,不属于某个用户;MVP 系统预置 key)
```

## DDL

### 账户组

```sql
-- sys_users(现有 · 保留;to C 下不再挂 role / department 关联,字段沿用现状)
CREATE TABLE sys_users (
    id VARCHAR(32) PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL,              -- bcrypt hash
    full_name VARCHAR(100),
    avatar VARCHAR(500),
    status VARCHAR(20) NOT NULL DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT chk_sys_users_status CHECK (status IN ('active', 'inactive', 'suspended'))
);
CREATE INDEX idx_sys_users_username ON sys_users(username);
CREATE INDEX idx_sys_users_email ON sys_users(email);
CREATE INDEX idx_sys_users_status ON sys_users(status);
-- 预留:日后区分平台管理员时加 is_admin BOOLEAN NOT NULL DEFAULT FALSE,不重建 role 表
```

### 模型组

```sql
-- ai_model_config(现有 · 保留;MVP 用系统预置 key,不计费。per-user BYOK 见文末后续扩展)
CREATE TABLE ai_model_config (
    id VARCHAR(32) PRIMARY KEY,
    vendor VARCHAR(32) NOT NULL,                 -- OpenAI / Anthropic / 硅基流动 等
    name VARCHAR(32) NOT NULL UNIQUE,            -- 配置名称
    api_key VARCHAR(256) NOT NULL,               -- 敏感,需加密存储
    base_url VARCHAR(256) NOT NULL,
    model_type VARCHAR(16) NOT NULL DEFAULT 'chat',  -- chat / embedding
    model VARCHAR(128) NOT NULL,
    temperature DECIMAL(3,2) NOT NULL DEFAULT 0.7,
    top_p DECIMAL(3,2) NOT NULL DEFAULT 0.9,
    max_tokens INTEGER,
    top_k INTEGER,
    is_active BOOLEAN NOT NULL DEFAULT TRUE,
    is_default BOOLEAN NOT NULL DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_ai_model_config_active ON ai_model_config(is_active);
CREATE UNIQUE INDEX idx_ai_model_config_type_default ON ai_model_config(model_type, is_default) WHERE is_default = TRUE;
```

### 学习核心组(新增)

```sql
-- 1. 学习目标(learning_goals)
CREATE TABLE learning_goals (
    id VARCHAR(32) PRIMARY KEY,
    user_id VARCHAR(32) NOT NULL REFERENCES sys_users(id) ON DELETE CASCADE,
    title VARCHAR(255) NOT NULL,                 -- 用户的一句话目标,如"我要学 Go 的 goroutine"
    description TEXT,                             -- 可选补充
    source VARCHAR(16) NOT NULL DEFAULT 'text',  -- text=一句话 / documents=上传材料(MVP 固定 text)
    status VARCHAR(20) NOT NULL DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT chk_learning_goals_source CHECK (source IN ('text', 'documents')),
    CONSTRAINT chk_learning_goals_status CHECK (status IN ('active', 'archived', 'completed'))
);
CREATE INDEX idx_learning_goals_user_id ON learning_goals(user_id);
CREATE INDEX idx_learning_goals_status ON learning_goals(status);
CREATE INDEX idx_learning_goals_created_at ON learning_goals(created_at DESC);

-- 2. 学习路线(learning_paths):一个目标一条路线
CREATE TABLE learning_paths (
    id VARCHAR(32) PRIMARY KEY,
    goal_id VARCHAR(32) NOT NULL UNIQUE REFERENCES learning_goals(id) ON DELETE CASCADE,
    user_id VARCHAR(32) NOT NULL REFERENCES sys_users(id) ON DELETE CASCADE,
    overview JSONB,                              -- 阶段大纲概要(stage 列表)
    current_objective_id VARCHAR(32),            -- 当前进行到的小目标(软引用,不加 FK 以避免循环依赖)
    status VARCHAR(20) NOT NULL DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT chk_learning_paths_status CHECK (status IN ('active', 'completed'))
);
CREATE INDEX idx_learning_paths_user_id ON learning_paths(user_id);

-- 3. 小目标(learning_objectives):路线里的节点,进度与掌握度的最小单位
CREATE TABLE learning_objectives (
    id VARCHAR(32) PRIMARY KEY,
    path_id VARCHAR(32) NOT NULL REFERENCES learning_paths(id) ON DELETE CASCADE,
    user_id VARCHAR(32) NOT NULL REFERENCES sys_users(id) ON DELETE CASCADE,
    stage_title VARCHAR(255),                    -- 所属阶段名
    title VARCHAR(255) NOT NULL,                 -- 小目标
    detail TEXT,                                 -- 小目标要点
    order_index INTEGER NOT NULL DEFAULT 0,      -- 路线内顺序
    status VARCHAR(20) NOT NULL DEFAULT 'pending',
    mastery_level VARCHAR(20) NOT NULL DEFAULT 'none',  -- 对齐 03 的掌握分级
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT chk_learning_objectives_status CHECK (status IN ('pending', 'active', 'completed', 'review')),
    CONSTRAINT chk_learning_objectives_mastery CHECK (mastery_level IN ('none','seen','heard','explained','written','verified'))
);
CREATE INDEX idx_learning_objectives_path_order ON learning_objectives(path_id, order_index);
CREATE INDEX idx_learning_objectives_user_id ON learning_objectives(user_id);
CREATE INDEX idx_learning_objectives_status ON learning_objectives(status);

-- 4. 学习会话(learning_sessions):一节课
CREATE TABLE learning_sessions (
    id VARCHAR(32) PRIMARY KEY,
    user_id VARCHAR(32) NOT NULL REFERENCES sys_users(id) ON DELETE CASCADE,
    goal_id VARCHAR(32) NOT NULL REFERENCES learning_goals(id) ON DELETE CASCADE,
    objective_id VARCHAR(32) NOT NULL REFERENCES learning_objectives(id) ON DELETE CASCADE,
    status VARCHAR(20) NOT NULL DEFAULT 'active',
    summary TEXT,                                -- 本节小结(Examiner 生成)
    message_count INTEGER DEFAULT 0,
    started_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    ended_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT chk_learning_sessions_status CHECK (status IN ('active', 'completed', 'abandoned'))
);
CREATE INDEX idx_learning_sessions_user_id ON learning_sessions(user_id);
CREATE INDEX idx_learning_sessions_goal_id ON learning_sessions(goal_id);
CREATE INDEX idx_learning_sessions_objective_id ON learning_sessions(objective_id);
CREATE INDEX idx_learning_sessions_created_at ON learning_sessions(created_at DESC);

-- 5. 陪练消息(learning_messages):对齐 ai_agent_messages,多加 agent_type 区分是哪个 agent
CREATE TABLE learning_messages (
    id VARCHAR(32) PRIMARY KEY,
    session_id VARCHAR(32) NOT NULL REFERENCES learning_sessions(id) ON DELETE CASCADE,
    role VARCHAR(20) NOT NULL,                   -- user / assistant / system
    agent_type VARCHAR(20),                      -- planner/tutor/examiner/orchestrator(assistant 消息标注)
    content TEXT NOT NULL,
    tool_used VARCHAR(100),
    tool_result JSONB,
    prompt_tokens BIGINT,
    completion_tokens BIGINT,
    total_tokens BIGINT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT chk_learning_messages_role CHECK (role IN ('user', 'assistant', 'system')),
    CONSTRAINT chk_learning_messages_agent CHECK (agent_type IS NULL OR agent_type IN ('planner','tutor','examiner','orchestrator'))
);
CREATE INDEX idx_learning_messages_session_id ON learning_messages(session_id, created_at);
CREATE INDEX idx_learning_messages_agent_type ON learning_messages(agent_type) WHERE agent_type IS NOT NULL;
CREATE INDEX idx_learning_messages_total_tokens ON learning_messages(total_tokens) WHERE total_tokens IS NOT NULL;
CREATE INDEX idx_learning_messages_tool_result ON learning_messages USING GIN (tool_result);

-- 6. 练习与验证(learning_exercises):"判断是否真学会"的结构化记录
CREATE TABLE learning_exercises (
    id VARCHAR(32) PRIMARY KEY,
    session_id VARCHAR(32) NOT NULL REFERENCES learning_sessions(id) ON DELETE CASCADE,
    objective_id VARCHAR(32) NOT NULL REFERENCES learning_objectives(id) ON DELETE CASCADE,
    user_id VARCHAR(32) NOT NULL REFERENCES sys_users(id) ON DELETE CASCADE,
    type VARCHAR(20) NOT NULL,                   -- explain/choice/cloze/paste_output/code_snippet
    prompt TEXT NOT NULL,                        -- 题目 / 要求
    user_answer TEXT,                            -- 用户作答(解释 / 代码 / 粘贴的运行结果)
    verdict VARCHAR(20),                         -- pass / partial / fail(Examiner 判定)
    mastery_after VARCHAR(20),                   -- 判定后的掌握层级
    feedback TEXT,                               -- 反馈
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT chk_learning_exercises_type CHECK (type IN ('explain','choice','cloze','paste_output','code_snippet')),
    CONSTRAINT chk_learning_exercises_verdict CHECK (verdict IS NULL OR verdict IN ('pass','partial','fail')),
    CONSTRAINT chk_learning_exercises_mastery CHECK (mastery_after IS NULL OR mastery_after IN ('none','seen','heard','explained','written','verified'))
);
CREATE INDEX idx_learning_exercises_session_id ON learning_exercises(session_id);
CREATE INDEX idx_learning_exercises_objective_id ON learning_exercises(objective_id);
CREATE INDEX idx_learning_exercises_user_id ON learning_exercises(user_id);

-- 7. 学习画像(learning_profiles):一个目标一份
CREATE TABLE learning_profiles (
    id VARCHAR(32) PRIMARY KEY,
    user_id VARCHAR(32) NOT NULL REFERENCES sys_users(id) ON DELETE CASCADE,
    goal_id VARCHAR(32) NOT NULL UNIQUE REFERENCES learning_goals(id) ON DELETE CASCADE,
    current_level VARCHAR(50),                   -- 当前水平概述
    completed_topics JSONB,                      -- 已掌握主题列表
    weak_points JSONB,                           -- 薄弱点列表
    verification_habits TEXT,                    -- 验证习惯
    next_goal TEXT,                              -- 下一步建议
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_learning_profiles_user_id ON learning_profiles(user_id);

-- 8. 学习日志(learning_journals):每天一条
CREATE TABLE learning_journals (
    id VARCHAR(32) PRIMARY KEY,
    user_id VARCHAR(32) NOT NULL REFERENCES sys_users(id) ON DELETE CASCADE,
    goal_id VARCHAR(32) NOT NULL REFERENCES learning_goals(id) ON DELETE CASCADE,
    date DATE NOT NULL,                          -- 学习日期
    learned TEXT,                                -- 学了什么
    evidence TEXT,                               -- 证据(验证记录)
    weak_points TEXT,                            -- 当日薄弱点
    next_step TEXT,                              -- 下一步
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT uq_learning_journal_day UNIQUE (user_id, goal_id, date)
);
CREATE INDEX idx_learning_journals_user_date ON learning_journals(user_id, date DESC);
CREATE INDEX idx_learning_journals_goal_id ON learning_journals(goal_id);
```

## 设计说明

- **掌握分级**(`mastery_level` / `mastery_after`)直接对齐 [03](03-学习闭环与教学法.md) 的 `none→seen→heard→explained→written→verified`,让"判断是否真学会"在数据层有落点。
- **多 agent 记录**:不另建 `agent_run` 表,用 `learning_sessions` + `learning_messages`(带 `agent_type`)承担,类比现有 `ai_agent_sessions/messages`。
- **token 字段**预留在 `learning_messages`,为后续用量计量 / 计费铺路。
- **`message_count`** 可沿用现有触发器模式自动维护(见 `ai/001_schema.sql` 的 trigger)。
- **路线结构**当前按线性(`order_index`);若改知识图谱(见 [07](07-决策记录与开放问题.md) Q4),`learning_objectives` 需加 `parent_id` / 依赖关系表。

## to C 精简:移除企业 RBAC

个人订阅用不到企业级 RBAC,现有 `system` 模块按下面处理(见 [07](07-决策记录与开放问题.md) D7):

- **保留**:`sys_users`、auth(登录认证)。
- **移除 4 张表**:`sys_departments`、`sys_roles`、`sys_user_departments`、`sys_user_roles`。
- **代码层(实现阶段清理)**:删除 `app/system` 下 role / department 的 handlers / models / repository / router;`system` migration 精简为只建 `sys_users` + 一个初始用户;注意 `middleware/auth.go` 若依赖 role 做鉴权,需改为简单的登录态校验。
