# 04 · Agent 系统设计

> 编排框架用 **eino**(在 Go 进程内),不另起 Python / JS 的 agent 服务。

## 角色总览

| Agent | 一句话职责 |
|---|---|
| 学习设计 agent(Planner) | 把"用户目标 + 资料"拆成学习路线 |
| 费曼教学 agent(Tutor) | 一节节陪学,每次只推一个认知点 |
| 学习监督 agent(Examiner) | 判断是否真学会,维护学习状态 |
| Learning Orchestrator | 决定何时调谁,控制 session 流程 |

## 4.1 学习设计 agent(Planner)

取代初版的"学科专家 agent"。**主题无关** —— 一个 agent 应对所有主题,而不是一个领域一个 agent。

- 输入:用户的一句话目标 +/或 上传的文档;
- 输出:一条学习路线(阶段 + 小目标序列);
- 职责:判断用户背景、安排学习顺序、拆分阶段目标、识别常见卡点、定第一阶段目标。

## 4.2 费曼教学 agent(Tutor)

通用陪练,执行 [03](03-学习闭环与教学法.md) 里的费曼教学循环。每次只推进一个认知点;先诊断后讲解;讲完给小练习并要求验证。

## 4.3 学习监督 agent(Examiner)

不是"总结对话",而是平台的**学习状态维护者**。

- 判断用户今天是否真掌握(对照 [03](03-学习闭环与教学法.md) 的掌握分级);
- 把学习事实写入 LearningProfile;
- 把每日记录写入 LearningJournal;
- 更新薄弱点、推荐下一步、标记需复习的概念。

它维护的是**学习状态机**,而非聊天记录。

## 4.4 Learning Orchestrator

- 判断当前该调用哪个 agent;
- 控制一个学习 session 的流程;
- 管理 agent 之间的输入输出;
- 避免多个 agent 同时对用户说话;
- 决定何时写日志、何时结束本节、下一步推荐什么。

没有它,多 agent 会变成一群角色同时聊天,体验会乱。

## 协作流程

```text
用户给出目标(一句话 或 上传文档)
        ↓
学习设计 agent 生成阶段路线
        ↓
Orchestrator 选择今天的一小节
        ↓
费曼教学 agent 陪学
        ↓
用户提交回答 / 运行代码
        ↓
监督 agent 判断学习结果
        ↓
写入学习画像 + 学习日志
        ↓
推荐下一步
```
