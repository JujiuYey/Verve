# 05 · 功能模块(MVP)

> 范围:**一句话主题驱动 + 技术/编程类**。本篇是 MVP 的功能模块详细规格。标 `[MVP外]` 的是后续阶段。
>
> 页面与路由见 [08-产品地图](08-产品地图.md),数据落点见 [09-表结构设计](09-表结构设计.md)。

## 模块总览

| 模块 | MVP 做什么 |
|---|---|
| A 账户与登录 | 注册 / 登录(复用 JWT) |
| B 学习目标 | 一句话创建、列表、详情、归档 |
| C 学习路线 | Planner 生成、阶段 / 小目标展示、进度 |
| D 陪练教学 | 单节课闭环(诊断→讲解→练习→验证),流式 |
| E 学习验证 | 技术编程类验证 + 掌握判定 |
| F 学习画像 | 只读展示,自动更新 |
| G 学习日志 | 每日自动生成 + 展示 |
| H 模型配置 | 系统预置 key(BYOK 后续) |

---

## A. 账户与登录

- 注册、登录、登出;复用现有 `sys_users` + JWT 中间件。
- MVP 单一普通用户角色,不启用 RBAC 的部门 / 角色管理。
- **验收**:能注册新账号、登录拿到 token、受保护接口校验通过。
- `[MVP外]` 第三方登录、找回密码、订阅状态。

## B. 学习目标

- **创建(一句话)**:输入"我要学 X" → 落 `learning_goals`(`source='text'`)→ 触发 C 的路线生成。
- **列表**:展示用户所有目标(active / 已归档)。
- **详情**:见 C(路线 + 进度)。
- **归档 / 完成**:改 `status`。
- **验收**:创建目标后能在总览看到卡片;能进入详情;能归档。
- `[MVP外]` 上传文档创建目标(`source='documents'`)、目标标签 / 分类。

## C. 学习路线

- **生成**:Planner agent 基于一句话目标产出阶段 + 小目标序列,落 `learning_paths` + `learning_objectives`。
- **展示**:阶段分组列出小目标,标出当前小目标(`current_objective_id`)与各自 `status` / `mastery_level`。
- **进度**:已完成 / 总数;当前学到哪。
- **验收**:提交目标后能看到一条结构化路线;当前小目标高亮;完成一节后进度推进。
- `[MVP外]` 用户手动调整路线(增删 / 改序)、agent 按表现自动重排(见 07 Q6)、知识图谱式路线(07 Q4)。

## D. 陪练教学(核心交互)

- **进入**:从目标详情"开始这一节"创建 `learning_sessions` → `/learn/:sessionId`。
- **单节流程**(Orchestrator 调度,见 [04](04-Agent系统设计.md)):诊断问题 → 简短讲解 → 小练习 → 要求验证 → 小结。
- **交互**:聊天式、流式输出(SSE → streamdown);每次只推进一个认知点(见 [03](03-学习闭环与教学法.md))。
- **消息留存**:落 `learning_messages`(带 `agent_type` 区分 tutor / examiner 等)。
- **结束**:`status=completed`,生成 `summary`,触发 F / G 写入。
- **验收**:能完整走完一节课,流式可见,过程被记录,结束后状态与画像 / 日志更新。
- `[MVP外]` 多目标并行 session、语音、富交互练习组件。

## E. 学习验证

- **验证类型**(技术编程类):`explain`(讲解)、`choice`(选择)、`cloze`(填空)、`paste_output`(粘贴本地运行结果)、`code_snippet`(提交最小代码片段)。
- **判定**:Examiner agent 判 `verdict`(pass/partial/fail)并给 `mastery_after`,落 `learning_exercises`,回写 `learning_objectives.mastery_level`。
- **原则**:不空泛鼓励;区分"能讲 / 能写 / 能验证"。
- **验收**:用户作答后能得到明确判定 + 反馈;掌握层级随之更新。
- `[MVP外]` 浏览器内运行代码、远程沙箱真实执行、自动测试。

## F. 学习画像

- **内容**:当前水平、已掌握主题、薄弱点、下一步;一个目标一份(`learning_profiles`)。
- **更新**:由 Examiner 在每节结束后自动写入;MVP 用户**只读**查看。
- **验收**:学完若干节后,画像能反映掌握主题与薄弱点的变化。
- `[MVP外]` 用户编辑画像(见 07 Q6)、跨目标的总画像。

## G. 学习日志

- **内容**:按天记录"学了什么 / 证据 / 薄弱点 / 下一步"(`learning_journals`,`(user,goal,date)` 唯一)。
- **生成**:Examiner 生成草稿;MVP 默认自动写入(确认式写入见 07 Q5)。
- **展示**:按天翻看。
- **验收**:每天学习后能看到当天日志;可翻历史。
- `[MVP外]` 用户编辑 / 补充日志、周报 / 月报聚合。

## H. 模型配置

- MVP **复用系统预置 `ai_model_config`**(平台 key,不计费),用户无需配置即可用。
- **验收**:陪练 / 验证调用走预置模型,正常流式返回。
- `[MVP外]` per-user BYOK(用户填自己的 key)、模型选择、用量与配额(商业化)。

---

## MVP 明确不做

- 文档 / 材料上传、解析、向量化(`Material` 链路)。
- 代码沙箱 / 远程执行。
- 订阅、计费、用量限额。
- to B:团队 / 成员 / 权限 / 部门。
