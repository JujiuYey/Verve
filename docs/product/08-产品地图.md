# 08 · 产品地图(MVP)

> 范围:**一句话主题驱动 + 技术/编程类**。本篇是 MVP 的页面、路由、导航与关键操作流。

## 页面与路由

```text
公共(无需登录)
  /login              登录
  /register           注册

应用(需登录,带主框架 + 侧边栏)
  /                   学习总览(我的目标 + 今天学什么 + 继续上次)
  /goals/new          新建学习目标(一句话输入)
  /goals/:goalId      目标详情(学习路线 + 进度 + 开始/继续)
  /learn/:sessionId   陪练教学(核心,聊天式单节课)
  /goals/:goalId/profile   学习画像(掌握度 / 薄弱点 / 下一步)
  /journal            学习日志(按天)
  /settings           设置(模型 / Key、账户)
```

## 导航结构(侧边栏)

```text
[Logo]
─ 学习总览        /
─ 我的目标        (目标列表)
     └ 目标 A / 目标 B …
─ 学习日志        /journal
─ 设置            /settings
[用户头像菜单:账户 / 登出]
```

## 页面职责

| 页面 | 核心内容 | 主要动作 |
|---|---|---|
| 学习总览 | 目标卡片、今日推荐、继续上次 | 进入目标 / 新建目标 / 继续学 |
| 新建目标 | 一句话输入框 + 引导示例 | 提交 → 触发路线生成 |
| 目标详情 | 学习路线(阶段 + 小目标)、进度、当前小目标 | 开始 / 继续这一节 → 进入陪练 |
| 陪练教学 | 聊天式对话、流式输出、练习卡、验证输入 | 回答 / 提交练习 / 粘贴结果 → 完成本节 |
| 学习画像 | 掌握分级、已完成、薄弱点、下一步 | 查看(MVP 只读) |
| 学习日志 | 按天的"学了什么 / 证据 / 薄弱点 / 下一步" | 查看 / 翻历史 |
| 设置 | 模型与 Key、账户信息 | 配置 |

## 关键操作流

### 流程一:新建目标 → 生成路线

```text
/goals/new 输入"我要学 X"
  → 提交,后端起 Planner agent(流式)
  → 生成学习路线(阶段 + 小目标)
  → 落库 learning_paths + learning_objectives
  → 跳到 /goals/:goalId,展示路线并高亮第一个小目标
```

### 流程二:上一节课(核心闭环)

```text
/goals/:goalId 点"开始这一节"
  → 创建 learning_sessions,进入 /learn/:sessionId
  → Orchestrator 调度,单节内部:
       诊断问题 → 简短讲解 → 小练习 → 要求验证
  → 用户提交(解释 / 选择 / 代码片段 / 粘贴运行结果)
  → Examiner 判定掌握层级,写 learning_exercises
  → 本节小结:更新 objective 状态 + learning_profiles + learning_journals
  → 推荐下一步(下一个小目标 / 复习)
```

### 流程三:次日继续

```text
/ 学习总览
  → "继续上次":读取 path.current_objective_id,直接进入下一节
```

## 与现有前端结构的映射(落地参考)

- **路由**:TanStack Router,挂在现有 `routes/_layout`(需登录布局)下,新增 learning 路由。
- **页面**:新建 `web/src/pages/learning/`,陪练界面复用现有 `components/ai-elements` + `streamdown`。
- **侧边栏**:复用现有 `layout/sidebar`,菜单项替换为学习平台导航。
- `(待深化)` 具体组件拆分、状态管理(zustand)、与后端 SSE 的对接。
